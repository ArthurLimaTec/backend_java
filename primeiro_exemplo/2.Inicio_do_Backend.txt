/* 
 * Como estruturar um backend:
 * 
 * Ele pode ser estruturado como uma APIRestful ou como micro serviços
 * 
 * -------------------------------------------------------
 * APIRestfull: possuem 'controllers' (recebem as requisições HTTP do Frontend), cada controller deve ser unitário (cuida de uma coisa só), ex: UsuarioController.
 *              os Controllers se comunicam com os 'Services'.
 *              O Repository é um serviço de persistência (responsável por guardar dados em algum lugar)
 *              Banco de dados: é a última etapa, onde os dados serão guardados.
 * 
 *                              APIRestfull
 * UsuarioController -> UsuarioService -> UsuarioRepository -> Banco de Dados
 * 
 * -------------------------------------------------------
 * Microserviços: possue 'Gatways' (recebe as requisições), que possuem microserviços diversos, que serão requisitados de acordo com a solicitação do cliente
 *                  dentro do Gatway, cada coisa do APIRestfull pode ser contido dentro de cada serviço do Gatway.
 *                  Ou seja, cada microserviço tem sua finalidade, faz, termina, devolve a resposta e segue para outro microserviço ou devolve resposta para o frontend
 *                  Cada microserviço pode ter um banco de dados, ou ser apenas um para todos. 
 * 
 * Cliente -> Frontend -> Gatway -> Microserviço 1   > Banco de Dados
 *                               -> Microserviço 2
 *                               -> Microserviço 3...
 * -------------------------------------------------------
*/
